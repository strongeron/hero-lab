Pixel Grid

Mosaic mask

Overlays the shader with a repeating grid of tiles — the "LED panel" mosaic look. This is the single most identity-defining system in the lab: it turns a smooth shader into crisp discrete tiles.

smooth shader → discrete tiles
Grid off vs on

Pixel Grid On/Off + Snap to dither

Core
pixelGrid.enabled · pixelGrid.snap

On masks the shader through a tile grid — the mosaic identity. Snap axis-aligns the dither and locks one dot per tile for crisp edges. Important nuance discovered in testing: snap flattens scene motion in Color/alpha mode (dots lock to grid, so drift/scan only flip tiles on/off). Turn snap off when you want scenes to visibly flow in Color/alpha.

Cuts & power verdict

The actionable takeaways: what to retire, and where the real strength of the system lives.

▶ Recommended cuts — the fastest way to simplify

Nine controls add surface without adding reach. Removing or folding them would make the panel noticeably calmer with zero loss of expressive range.

Remove / fold

  • Tile Animation (duplicate) — same control in both Pixel Grid and Animation; keep one.
  • Multi-Color — Alpha Layers already give 3 colors + shapes with more control.
  • Cell Size — inert while Pixel Grid is on (always). Hide when grid enabled.
  • Frame scrubber — manual still-frame; no place in a live playground.
  • World Width / Height — pattern aspect; niche to the point of unused.

Collapse / gate

  • Haptics — touch-only; hide or fold behind a "Touch" section on desktop.
  • Scene presets — 11 → ~7; merge Orbit/Living, Breathe/Breath, Color Flow/Gentle Drift.
  • Baseline guides — a debug aid; move under a debug affordance.
  • Fit / Rotation / Offset / Origin — keep, but fold into an "advanced sizing" group.

The load-bearing spine

  • Initial State — the Problem ⇄ Fix premise the whole hero rests on.
  • Pixel Grid geometry — tile size, divide, gap, radius: the mosaic identity.
  • Tile Animation modes — three renderers; the biggest look lever after the grid.
  • Section Edge — six modes + scene-behind-text; turns background into layout.
  • Hover warp-reveal — the signature "peek the other state" interaction.

Where the power really is

  • Any look (grid geometry + tile mode + colors) composes freely with any motion (scene) and any edge treatment — three orthogonal axes multiply into a very large design space.
  • The same config drives desktop, tablet and mobile live-synced, so every combination is verified responsive by construction.
  • The hover warp-reveal + error text is a genuinely distinctive interaction few devtool heroes have — this is the showcase moment.

One sharp edge to remember

  • Snap + Color/alpha flattens scene motion — dots lock to the grid, so drift/scan only twinkle. For visible scene motion in Color/alpha, turn snap off; for crisp static grids, keep it on. This single interaction caused most of the "scenes don't do anything" confusion.
